# PRD: 付费文章解锁 (feat-001)

## 背景

用户想基于 qiaomu-blog-opensource 开源博客模板搭建一个出售电子资料的网站。目标用户以国内为主，使用 Stripe（含微信支付）收款。

## 核心需求

1. **文章即产品**：每篇博客文章 = 一个可售卖产品，在后台编辑时设定价格和交付链接
2. **付费墙**：未付款访客看不到交付链接，看到价格 + 购买入口
3. **支付解锁**：付款后页面上链接解锁显示 + 邮件兜底发送链接
4. **店铺模式列表**：首页只展示付费文章，卡片带价格徽章
5. **合规与争议防护**：退款政策页 + 购买时强制勾选同意 + Checkout/邮件再声明 + Cookie GDPR 横幅
6. **运营追踪**：GA4 / Clarity / Yandex / GSC / Bing Webmaster

## 用户故事

### 买家视角
- 访客打开产品页 → 看到介绍内容和价格 → 填邮箱 + 勾选退款政策 → 点购买 → 跳 Stripe 付款 → 付款后回到页面看到交付链接 → 同时收到邮件
- 关掉页面后通过邮件里的解锁链接重新访问

### 卖家视角
- 在博客后台写文章，填价格和交付链接 → 发布 → 自动出现在首页列表
- 不想卖的文章把价格设为 0 → 从首页消失（直链仍可访问）
- 退款争议时，有勾选记录 + 邮件证据 + Checkout 声明三重证据

## 不做的（YAGNI）

- 用户注册 / 登录 / 个人中心
- 购物车 / 多商品合并下单
- 独立产品管理模块
- 文件直传 R2
- 多语言 / 多货币切换
- 优惠券 / 促销 / 退款后台
- 全功能 Cookie 颗粒同意管理

## 风险

| 风险 | 影响 | 缓解 |
|------|------|------|
| Stripe 账户国内个人难开 | 无法收款 | 假设用户已解决；如未解决需用港/美/新主体 |
| 电子资料售后争议 | 退款损失 | 三重证据链：勾选记录 + Checkout 声明 + 邮件留档 |
| 交付链接被分享 | 未付费者获取 | MVP 不做防盗链；可后续加一次性链接/R2 签名 URL |
| Cloudflare Workers 限制 | 执行超时 | 当前方案远在限制内 |
